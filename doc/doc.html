Initializes the Flapjax framework, returning an object whose fields are the functions defined below. If no arguments are specified, all Flapjax functions are exported to window if a function is already defined, it is not exported. If False is supplied, nothing is exported. (However, all of Flapjax is still accessible from the returned object.)

Flapjax's functions may be selectively exported as well. If includeSynonyms is set to False, synonyms for methods are not exported. exportMisc, exportCore, exportDOM, exportDOMMisc or exportWS are set, categories of functions can be selectively exported.

By default, Flapjax will not export functions that are already defined. For example, the Prototype Framework defines $, $A, etc. Flapjax will not redefine these functions. However, if redefine is set, Flapjax will overwrite Prototype's bindings.

Finally, hide and show can be used to selectively export specific functions.

getWebServiceObjectE(requestE) -> EventStream response requestE = EventStream request request = { url: string, request: 'get' | 'rest' | 'post' | 'rawPost', response: 'json', 'xml', 'plain', fields: object | undefined, body: string | undefined asynchronous: bool }

Sends a stream of requests to a web server and receives a stream of responses.

url is the address of the web service. getWebServiceObject_e is based on XMLHttpRequest, so the service must be on the same server as your application.

request specifies how the request should be encoded.

  • In a get request, each field in fields is encoded as a GET parameter. The value of each field is cast to a string.
  • A rest request is similar to a get request. Each field is encoded as a POST parameter. The value of each field is cast to a string.
  • In a post request, the fields object is serialized as a JSON string and sent as the body of the request.
  • In a rawPost request, the body string is sent as the body of the request.

response specifies how the response is decoded. plain indicates that the body should not be decoded. This is the default behavior. json indicates that the response should be parsed as a JSON string. xml indicates that the response is XML.
